    Asymptotics¶

    Asymptotic pronaunce "(a)Sym`Totic"

    Asymptotic Analysis¶

    We need to learn how to make the best use of our computer's time and memory.

    We can consider the process of writing efficient programs from two different perspectives:

    1. Programming Cost (everything in the course up to this date)
      1. How long does it take for you to develop your programs?
      2. How easy is it to read or modify your code?
      3. How maintainable is your code? (very important — much of the cost comes from maintenance and scalability, not development!)
    2. Execution Cost (everything in the course from this point on)
      1. Time complexity: How much time does it take for your program to execute?
      2. Space complexity: How much memory does your program require?

    Notice: asymptotics work for very long time, therefore 100 times is not a large number of time. Which means, if f(N) belongs to Theta(N^2), then input size of 2N will run roughly 4 times as long input size of N. BUT, if input size is 100, then input size of 200 will NOT run 4 times because 100 and 200 are all very small numbers.

    Cost Model¶

    在分析算法的运行时间时，我们通常会考虑算法执行过程中涉及的各种操作。然而，计算每一个具体操作的成本可能过于复杂。因此，简化分析的方法之一是选择一个有代表性的操作，作为衡量整体运行时间的基准。这种方法帮助我们简化复杂的算法分析。

    代表性操作的选择:

    好的选择： 1. 自增操作（increment）：如i++或i += 1。这是一个常见且简单的操作，通常在循环中出现。 1. 比较操作（less than or equals）：如i <= n或i < n。比较操作在条件判断中很常见。 1. 数组访问操作（array accesses）：如arr[i]。访问数组元素是很多算法中的基础操作。 这些操作之所以是好的选择，是因为它们在很多算法中频繁出现，并且它们的执行时间相对稳定且可以很好地反映算法的整体复杂度。

    不好的选择： 1. 赋值操作（assignment）：如j = i + 1或i = 0。这些操作通常只执行一次或少量几次，不能很好地代表算法的整体运行时间。赋值操作之所以不是好的选择，是因为它们通常不会在算法的核心循环或主要逻辑中频繁出现，因此不能有效代表算法的整体运行时间。

    成本模型

    所选择的代表性操作被称为“成本模型”。成本模型的选择是为了简化算法的复杂度分析，使我们能够专注于算法中最能反映其性能的部分。通过仅关注一个代表性操作，我们可以更简洁地估算和比较不同算法的效率。

    示例:在一个寻找重复元素（dup）的示例中： * 好的选择：选择自增操作i++或数组访问操作arr[i]作为成本模型，可以帮助我们评估算法在每次循环迭代中的基本操作次数，从而估算整体运行时间。 * 不好的选择：选择赋值操作j = i + 1或i = 0作为成本模型，因为这些操作只执行一次或少量几次，不能有效代表算法的整体运行时间。 通过选择合适的成本模型，我们可以更直观地分析和比较算法的性能。
