    Mkdocs and github

    主要内容¶

    md文件：记录、备忘录等 mkdocs：把md文件生成为静态网页 material for mkdocs：mkdocs的一个主题 github：代码托管网站 github page：免费的静态网页，可以存放mkdocs

    常规流程：生成静态网页¶

    1. 创建一个用来生成静态网页的项目 cd .... 目录转移到临时的文件夹 mkdocs new my-project cd my-project
    2. 打开生成的mkdocs.yml文件，修改内容为自动生成目录，例如：
    site_name: Jeff Liu Lab's wiki

# 配置Material主题
theme:
  name: 'material'

plugins:
  - search # MkDocs内置搜索插件
  - awesome-pages
    1. 将md文件和文件夹复制到目录内的docs文件夹下
    2. 使用mkdocs生成静态网页 mkdocs build 生成好的静态文件在site文件夹下
    3. 使用本地服务器检查 mkdocs serve 这将启动一个本地开发服务器，通常会在 http://127.0.0.1:8000 提供实时预览。每当修改文档时，页面将自动刷新。
    4. 将静态网页同步到github page 将site文件夹内的文件同步到github page相应的项目中 (如果忘掉github如何同步，参见github相关文档)

    安装mkdocs¶

    1. 安装mkdocs：pip install mkdocs
    2. 使用 mkdocs new 命令创建一个新的 MkDocs 项目： mkdocs new my-project cd my-project 这将在当前目录下创建一个名为 my-project 的文件夹，并在其中生成一些初始文件和文件夹。
    3. 项目目录结构大致如下： my-project/ docs/ index.md mkdocs.yml docs/ 目录：存放所有的 Markdown 文件，即项目文档内容。 index.md 文件：默认的主页文档。 mkdocs.yml 文件：项目的配置文件。
    4. 在项目目录中，运行以下命令启动 MkDocs 开发服务器： mkdocs serve 这将启动一个本地开发服务器，通常会在 http://127.0.0.1:8000 提供实时预览。每当修改文档时，页面将自动刷新。
    5. 配置项目 配置文件 mkdocs.yml 用于自定义站点的行为和外观。
    6. 构建站点 当你完成了所有的文档编写，可以运行以下命令生成静态网站文件： mkdocs build 生成的静态文件将位于 site 目录中。

    自动生成目录插件¶

    1. 安装mkdocs-awesome-pages-plugin： pip install mkdocs-awesome-pages-plugin
    2. 使用方法 在 mkdocs.yml 文件中，添加 mkdocs-awesome-pages-plugin 插件配置：
    site_name: My Project
theme: readthedocs

plugins:
  - search
  - awesome-pages

    MkDocs 默认情况下是按照文件系统中的顺序来生成导航菜单的，这通常是字母顺序。awesome-pages 插件也遵循这一规则，除非你明确指定了排序顺序。

    注意：1和10在mkdocs中是同样优先的，因此请用01和10来表示顺序。
