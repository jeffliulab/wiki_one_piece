    LinkedList

    java内置LinkedList

    import java.util.LinkedList;

public class Main {
    public static void main(String[] args) {
        // 创建一个 LinkedList 实例
        LinkedList<Integer> list = new LinkedList<>();

        // 添加元素到链表
        list.add(5);
        list.add(10);
        list.add(15);

        // 打印链表中的元素
        System.out.println("LinkedList: " + list);

        // 获取链表的长度
        System.out.println("Length: " + list.size());

        // 检查是否包含某个元素
        System.out.println("Contains 10: " + list.contains(10));
        System.out.println("Contains 20: " + list.contains(20));

        // 移除链表中的元素
        list.remove(Integer.valueOf(10));
        System.out.println("LinkedList after removing 10: " + list);

        // 遍历链表中的元素
        System.out.print("LinkedList elements: ");
        for (int element : list) {
            System.out.print(element + " ");
        }
    }
}

    使用内置的linkedlist的好处： 简洁方便：提供了许多内置的方法用于常见的操作，如添加、删除、访问元素等。 双向链表：LinkedList 是一个双向链表，实现了 Deque 接口，支持在两端添加和删除元素。 性能：适用于频繁的插入和删除操作，因为这些操作在链表中的时间复杂度为 O(1)。

    自定义的IntList

    public class IntList {
    public int value;
    public IntList next;

    public IntList(int value, IntList next) {
        this.value = value;
        this.next = next;
    }

    public int size(){
        IntList x = this;
        int i = 1; //or int i = 0, then use x != null as the while condition
        while(x.next != null){
            i++;
            x = x.next;
        }
        return i;
    }

    public void printList(){
        IntList x = this; //在 Java 中，this 不能被重新赋值，因此使用局部变量来遍历链表是必要的。
        while(x != null){
            System.out.print(x.value+" ");
            x = x.next;
        }
        System.out.println();
    }

    public int sizeR(){
        if(this.next == null){
            return 1;
        }
        else{
            return 1 + this.next.sizeR(); //这里的this.next实质上已经是一个instantiate了，也就是一个object，所以可以直接call .sizeR()
        }
    }

    public int get(int i){
        //Be sure the object you call is the first node，因为现在创建的方式比较原始
        //index i starting from 0
        IntList l = this;
        for(int j = 0 ; j < this.size() ; j++){
            if(i == j){
                return l.value;
            }
            l = l.next;
        }
        return l.value;
    }
}

    自定义的SLList

    IntList的问题：在IntList的实现中，链表是一个裸露的递归数据结构（naked recursive data structure）。IntNode 类直接表示链表节点，并且 next 指向下一个节点。用户需要直接操作这些节点和递归结构。例如，要打印链表中的所有节点，需要编写递归或迭代代码：

    为了简化用户对递归数据结构的操作，我们通常会在这些数据结构的基础上提供更高级的封装和抽象。例如，使用一个封装类 SLList 来隐藏递归实现细节，并提供更易用的方法。

    public class SLList {
    private Node first_node;
    public SLList(int i){
        first_node = new Node(i, null);
    }
    public void add_first_node(int i){
        first_node = new Node(i, first_node);
    }
    public Node get_first_node(){
        return first_node;
    }

    public void add_last_node(int i){
        //firstly find the last node now
        Node x = this.get_first_node();
        while(x.getNext()!=null){
            x = x.next;
        }
        x.next = new Node(i, null);
    }

    public int size(){
        Node x = this.get_first_node();
        int count = 1;
        while(x.getNext()!=null){
            x = x.next;
            count++;
        }
        return count;
    }

    public int sizeR(Node n){ //sizeR可以看做是sizeRR的helper method
    //sizeR的作用是封装和简化递归计算逻辑，从而使sizeRR() 更加简洁易懂。
        if(n.next==null){
            return 1;
        }
        else{
            return 1 + sizeR(n.next);
        }
    }

    public int sizeRR(){ //Due to Overloaded, here sizeRR can name as sizeR.
        return sizeR(first_node);
    }

    public static class Node{ //Node更像是一个helper class，因此用static嵌套在SLList内
        private int item;
        private Node next;
        public Node(int i, Node n){
            item = i;
            next = n;
        }
        public int getItem(){
            return item;
        }
        public Node getNext(){
            return next;
        }
    }
}

    This modification makes our size method incredibly fast, no matter how large the list. Of course, it will also slow down our addFirst and addLast methods, and also increase the memory of usage of our class, but only by a trivial amount. In this case, the tradeoff is clearly in favor of creating a cache for size.

    Improve6: Sentinel Nodes

    This solution works, but special case code like that shown above should be avoided when necessary. Human beings only have so much working memory, and thus we want to keep complexity under control wherever possible. For a simple data structure like the SLList, the number of special cases is small. More complicated data structures like trees can get much, much uglier.

    A cleaner, though less obvious solution, is to make it so that all SLLists are the "same", even if they are empty. We can do this by creating a special node that is always there, which we will call a sentinel node. The sentinel node will hold a value, which we won't care about.

    Invariants An invariant is a fact about a data structure that is guaranteed to be true (assuming there are no bugs in your code).

    A SLList with a sentinel node has at least the following invariants:

    The sentinel reference always points to a sentinel node.

    The front item (if it exists), is always at sentinel.next.item.

    The size variable is always the total number of items that have been added.

    So after above updating, the final SLList would be like:

    public class SLList {
    private Node sentinel;
    private Node first_node;
    private int size;

    public SLList(){
        sentinel = new Node(42, null);
        first_node = null;
        size = 0;
    }

    public SLList(int i){
        first_node = new Node(i, null);
        sentinel = new Node(42, first_node);
        size = 1;
    }

    public Node getFirst_node(){
        return first_node;
    }

    public int getNode(int index){

        if(index+1>size){
            System.out.println("Error: out of bounds.");
            return -1;
        }
        Node x = sentinel;
        if(x.getNext()==null){
            System.out.println("Error: no items in the linked list.");
            return -1;
        }

        int now_index = -1;

        while(x.getNext()!=null){
            if(now_index==index){
                return x.getItem();
            }
            x = x.getNext();
            now_index++;
        }
        return now_index;
    }

    public void add_first_node(int i){
        SLList x = this;
        first_node = new Node(i, first_node);
        sentinel = new Node(42, first_node);
        size++;
    }

    public void add(int i){
        Node x = sentinel;
        if(sentinel.getNext()==null){
            add_first_node(i);
        }
        else{
            while(x.getNext()!=null){
                x = x.getNext();
            }
            x.Special_Change_Next(i);
            size++;
        }
    }

    public int size(){
        return size;
    }

    public static class Node{
        private int item;
        private Node next;
        public Node(int i, Node n){
            item = i;
            next = n;
        }
        public int getItem(){
            return item;
        }
        public Node getNext(){
            return next;
        }
        public void Special_Change_Next(int i){
            next = new Node(i, null);
        }
    }
}

    自定义的DLList

    Improve7: Looking back: .last and .prev allow fast removeLast.

    public class DDList{
    private Node t0;
    private Node firstNode;
    private Node lastNode;
    private Node tN;
    private int size;

    //constructor, no arguments (create empty list)
    public DDList(){
        t0 = new Node(null,42,null);
        tN = new Node(null, 98, null);
        t0.next = tN;
        tN.prev = t0;
        firstNode = null;
        lastNode = null;
        size = 0;
    }

    //constructor, with arguments (create list with first node)
    public DDList(int i){
        t0 = new Node(null,42,null);
        tN = new Node(null, 98, null);
        Node x = new Node(t0, i, tN);
        t0.next = x;
        tN.prev = x;
        firstNode = x;
        lastNode = x;
        size = 1;
    }

    //getFirst()
    public Node getFirst(){
        return firstNode;
    }

    //getLast()
    public Node getLast(){
        return lastNode;
    }

    //get(index)
    public int get(int index){
        if(size==0){
            System.out.println("Exception: no item in the list.");
            return -1;
        }
        else if(index > size-1){
            System.out.println("Exception: index out of range.");
            return -1;
        }
        else {
            Node x = this.getFirst(); //now x is index 0, if return x, get List(0)
            int index_now = 0;
            while(index_now < index){
                x = x.next;
                index_now++;
            }
            return x.item;
        }
    }

    //getSize
    public int getSize(){
        return size;
    }

    //addFirst
    public void addFirst(int i){
        if(size==0){
            Node x = new Node(t0, i, tN);
            t0.next = x;
            tN.prev = x;
            firstNode = x;
            lastNode = x;
            size++;
        }
        else{
            Node x = new Node(null, i, null);
            x.next = firstNode;
            x.prev = t0;
            t0.next = x;
            firstNode.prev = x;
            firstNode = x;
            size++;
        }

    }

    //addLast
    public void addLast(int i){
        if(size==0){
            Node x = new Node(t0, i, tN);
            t0.next = x;
            tN.prev = x;
            firstNode = x;
            lastNode = x;
            size++;
        }
        else{
            Node y = new Node(null,i,null);
            y.next = tN;
            y.prev = lastNode;
            lastNode.next = y;
            tN.prev = y;
            lastNode = y;
            size++;
        }
    }

    public static class Node{
        public int item;
        public Node next;
        public Node prev;
        public Node(Node p, int i, Node n){
            item = i;
            next = n;
            prev = p;
        }
    }
}

    public class DDList<T>{

    private Node<T> firstNode;
    private Node<T> lastNode;
    private Node<T> tN;

        t0.next = tN;
        tN.prev = t0;
        firstNode = null;
        lastNode = null;
        size = 0;
    }

    //constructor, with arguments (create list with first node)
    public DDList(T i){
        t0 = new Node<T>(null,null,null);
        tN = new Node<T>(null, null, null);
        Node<T> x = new Node<T>(t0, i, tN);
        t0.next = x;
        tN.prev = x;
        firstNode = x;
        lastNode = x;
        size = 1;
    }

    //getFirst()
    public Node<T> getFirst(){
        return firstNode;
    }

    //getLast()
    public Node<T> getLast(){
        return lastNode;
    }

    //get(index)
    public T get(int index){

            return null;

    }

    public int getSize(){
        return size;
    }

            tN.prev = x;
            firstNode = x;

            size++;

        else{
            Node<T> x = new Node<T>(null, i, null);
            x.next = firstNode;
            x.prev = t0;
            t0.next = x;
            firstNode.prev = x;
            firstNode = x;
            size++;
        }

    }

        if(size==0){

            t0.next = x;
            tN.prev = x;
            firstNode = x;
            lastNode = x;
            size++;
        }
        else{
            Node<T> y = new Node<T>(null,i,null);
            y.next = tN;
            y.prev = lastNode;
            lastNode.next = y;
            tN.prev = y;
            lastNode = y;
            size++;
        }
    }

    public static class Node<T>{
        public T item;
        public Node<T> next;
        public Node<T> prev;
        public Node(Node<T> p, T i, Node<T> n){
            item = i;
            next = n;
            prev = p;
        }
    }
}
